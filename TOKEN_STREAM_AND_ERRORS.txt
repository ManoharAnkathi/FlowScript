FlowScript Quick Guide: Tokens, Keywords, and Errors

What this file helps you with
- Where to put input code
- What each keyword means
- How code becomes a stream of tokens
- What common errors look like

1) Input file to run
- Main demo file: sample.flow

Run commands:
- Tokens only:
  /usr/local/bin/python3 main.py sample.flow --phase tokens --token-limit 400
- Parser (syntax):
  /usr/local/bin/python3 main.py sample.flow --phase parse --show-ast
- Semantic checks:
  /usr/local/bin/python3 main.py sample.flow --phase semantic --show-symbols

2) What each keyword/directive does

Program boundaries
- @start: starts the program
- @end: ends the program

Variables and output
- let: declare a variable
  Example: let a := 5
- const: declare a constant (cannot be reassigned)
  Example: const pi := 3
- print => expr: prints a value/expression
  Example: print => a

Conditions and loops
- @if condition ... @endif: conditional block
  Example:
    @if a > b
    print => a
    @endif

- @repeat n ... @endrepeat: repeat block n times (front-end support)
  Example:
    @repeat 3
    print => a
    @endrepeat

- @for i := start to end ... @endfor: range loop
  Example:
    @for i := 1 to 5
    print => i
    @endfor

- @while condition ... @endwhile: while loop
  Example:
    @while count > 0
    print => count
    @endwhile

Functions and flow control
- @function name ... @endfunction: function declaration
  Example:
    @function show
    print => 1
    @endfunction

- return expr: return from function (semantic check: only inside function)
  Examples:
    return value
    return => value
  Alias:
    giveback

- continue: skip to next loop iteration (only inside loop)
  Alias:
    skip

- break: exit current loop (only inside loop)
  Alias:
    stop

3) How code becomes a token stream
- The lexer reads input line by line.
- It recognizes words/symbols and emits tokens in order.
- Example line:
  let a := 5
- Token stream for that line:
  LET, IDENT(a), ASSIGN(:=), NUMBER(5)

4) Common symbols
- := assignment
- => print/return arrow form
- + - * / arithmetic operators
- > comparison operator
- ( ) grouping

5) Common error files and what to expect
- Folder: error_cases

Lexical error
- File: error_cases/lex_unexpected_character.txt
- Expect: Unexpected character error (example: '$')

Parser errors
- File: error_cases/parse_missing_start.txt
  Expect: Program must start with @start
- File: error_cases/parse_missing_end.txt
  Expect: Program must end with @end
- File: error_cases/parse_bad_print.txt
  Expect: Expected '=>' after 'print'
- File: error_cases/parse_unknown_directive.txt
  Expect: Unknown directive with suggestion
- File: error_cases/parse_bad_for_header.txt
  Expect: Expected 'to' in for-loop header

Semantic errors
- File: error_cases/semantic_const_reassign.txt
  Expect: Cannot reassign constant
- File: error_cases/semantic_return_outside_function.txt
  Expect: 'return' is only allowed inside a function
- File: error_cases/semantic_continue_outside_loop.txt
  Expect: 'continue' is only allowed inside a loop
- File: error_cases/semantic_break_outside_loop.txt
  Expect: 'break' is only allowed inside a loop
